# Principles

## Write for repo browsing first

Assume someone is reading the folder tree directly in a Git host or text editor.

## Use explicit section headings

Predictable section labels make the archive easier to scan and easier for AI to transform.

## Keep metadata generic

Avoid fields that only matter to one framework or theme.

## Prefer clear filenames

Filenames should make sense in a terminal listing without needing additional UI context.
